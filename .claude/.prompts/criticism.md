# Criticism — Adversarial Strategic Review

Deep adversarial analysis of the project. Unlike /explore (divergent) or /cleanup (reconciliation), /criticism actively looks for what's broken, fragile, missing, misaligned, or wasted.

## Protocol

1. Load full project context (brain.db, active plans, recent sessions)
2. Analyze through 5 lenses: Architecture, Decisions, Direction, Assumptions, Waste
3. Rate each finding: CRITICAL / HIGH / MEDIUM / LOW
4. Deliver structured report with actionable remediation for each finding
5. Record significant findings to brain.db

See `.claude/skills/criticism/SKILL.md` for the complete analysis framework.
