# Knowledge Curator - INGEST

## Insert Paths

There are three ways knowledge enters the system. The curator agent is involved in path 3.

| Path | Trigger | Vector (ChromaDB) | Vault filing | GRAPH/INDEX | Agent |
|---|---|---|---|---|---|
| **Quick Insert** | Insert tab → Quick Add | ✅ direct | ❌ | ❌ | ❌ |
| **Paste → Ingest All** | Insert tab → Ingest All | ✅ `/ingest-file` | ✅ auto-filed | ❌ | ❌ |
| **Agent process** | Agent tab → "/process" | ✅ via API call | ✅ | ✅ | ✅ curator |

- **Quick Insert** is a fast-capture path (term = definition → ChromaDB). No file, no vault, no agent.
- **Paste → Ingest All** saves markdown to `new/`, then the server handles vector ingestion + vault filing + archiving automatically.
- **Agent process** is the full-enrichment path: the curator reads files from `new/`, validates, enriches, extracts entities/relationships, updates GRAPH.md and INDEX.md, calls the embedding API for vector ingestion, files to vault, and archives.

## Expected Input Format

Cheat sheets generated by Learning_Session_Configuration.md arrive with:

**Frontmatter (required fields):**
```yaml
domain: cheatSheets
category: Python|DataScience|Automation|Tools|Linux|General
tags: [topic-tag, learning]
title: Descriptive Title
created: YYYY-MM-DD
session: Session Topic
status: new
```

**Body sections:**
- `## ⚡ Quick Reference` — Key:Value syntax recall
- `## 🧠 Functional Logic` — Concepts with [[wikilinks]]
- `## 💻 Implementation` — Copy-pasteable code
- `## 🕸️ Graph Connections` — Parents / Children / Lateral links (pre-structured by the author)
- `## 🛠️ Sandbox / To Explore` — Open questions

## Agent Workflow

When user says "process cheat sheets" (via Agent tab or direct message):

1. **Scan**: List files in source directory (`/home/ava/Ava_Main/repos/cheatSheets/new/`)
2. **Read**: Get full content of each file
3. **Validate Frontmatter**: Check for required fields (`domain`, `category`, `tags`, `title`, `created`, `status`)
   - If `category` is missing: infer from content using Category Rules below
   - If `status` is not `new`: skip (already processed)
   - If other fields missing: add sensible defaults, flag in report
4. **Analyze Vault State**: Read GRAPH.md and INDEX.md to understand existing content
5. **Categorize**: Confirm or correct the `category` field using Category Rules
6. **Enhance Tags**: Ensure tags are lowercase, hyphenated, and include `learning`; add any missing topic tags
7. **Extract Entities**: From body content, identify:
   - `concept` — what it teaches (e.g., variance, recursion)
   - `library` — packages referenced (e.g., NumPy, Pandas)
   - `function` — specific methods demonstrated (e.g., np.var, pd.DataFrame)
   - `formula` — mathematical expressions (e.g., σ² = Σ(Xi - μ)² / N)
   - `technique` — patterns described (e.g., Z-score normalization)
   - `tool` — tools used (e.g., git, docker)
8. **Extract Relationships**: From `## 🕸️ Graph Connections` section AND body [[wikilinks]]:
   - Parse existing Parents → `belongs_to` relationships
   - Parse existing Children → `parent_of` relationships
   - Parse existing Lateral → `related_to` relationships
   - Cross-reference against GRAPH.md entities for additional connections:
     - `implements` — code realizing a concept
     - `prerequisite_for` — must understand A before B
     - `derived_from` — mathematically/logically derived
     - `used_in` — applied within a technique
     - `alternative_to` — serves similar purpose
     - `extends` — builds upon
9. **Enrich Graph Connections**: Update the `## 🕸️ Graph Connections` section in the cheat sheet with any new relationships discovered in step 8 (add to existing, don't replace)
10. **Update Status**: Change `status: new` → `status: processed` in frontmatter
11. **Vector Ingest**: Call the embedding API to store in ChromaDB:
    ```bash
    curl -X POST http://127.0.0.1:8001/ingest-file \
      -H "Content-Type: application/json" \
      -d '{"file_path": "/home/ava/Ava_Main/repos/cheatSheets/new/FILENAME.md"}'
    ```
    Verify response has `"ok": true` before proceeding.
12. **Copy to Vault**: To `Knowledge/{category}/` folder
13. **Update GRAPH.md**: Append new entities and relationships, update counts, add clusters if 3+ tightly related entities emerge
14. **Update INDEX.md**: Add entry using the `title` frontmatter field
15. **Archive**: Move original to processed directory

## Category Rules

| Category | Signal |
|---|---|
| **Python** | Has code with `import`, `def`, `np.`, `pd.`, code blocks with Python syntax |
| **DataScience** | Theory-focused — algorithms, concepts, formulas, no/minimal code |
| **Automation** | Testing frameworks, CI/CD, workflows, agents, orchestration |
| **Tools** | CLI usage, configuration files, installation steps |
| **Linux** | Shell scripting, systemd, networking, permissions, kernel, OS-level |
| **General** | Cross-domain, meta-learning, doesn't fit above categories |

**Disambiguation:**
- Code + theory → `Python` (code wins)
- Shell script teaching bash syntax → `Linux`
- Shell script automating a CI pipeline → `Automation`
- Docker CLI reference → `Tools`; Docker-based deployment workflow → `Automation`

## Validation Errors (stop processing, report)
- File has no frontmatter at all
- File has `status: processed` (already done)
- File body is empty
- Vector ingest failed (do not archive — leave in `new/` for retry)
