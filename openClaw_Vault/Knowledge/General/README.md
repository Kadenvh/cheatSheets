# Knowledge/General

Notes that span multiple domains or don't fit neatly into other categories.

## Category Definition

A note belongs in **General/** if it covers:
- Cross-cutting concepts (design patterns, architecture principles)
- Meta-learning (study techniques, documentation practices)
- Mixed-domain topics without a clear primary category
- Soft skills applied to technical work (debugging strategies, problem-solving frameworks)

## When to Use General

Use **General/** only when a note genuinely doesn't fit elsewhere. If it leans toward one category, put it there instead.
