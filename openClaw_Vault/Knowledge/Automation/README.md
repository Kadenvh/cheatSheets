# Knowledge/Automation

Testing, CI/CD, workflows, and agent development notes focused on automated processes and system orchestration.

## Category Definition

A note belongs in **Automation/** if it covers:
- Testing frameworks (`pytest`, `unittest`, test patterns)
- CI/CD pipelines (GitHub Actions, Jenkins, deployment)
- Workflow automation and orchestration
- Agent development (OpenClaw skills, autonomous systems)
- Build and deployment automation

## Typical Contents

- Testing patterns and best practices
- pytest, unittest, integration testing
- GitHub Actions workflows
- Jenkins pipelines
- OpenClaw skill development
- Autonomous agent patterns
- Deployment automation

## Example Notes

- **pytest_fixtures.md** - pytest setup and teardown patterns
- **github_actions_ci.md** - CI/CD pipeline configuration
- **openclaw_skills.md** - Creating custom OpenClaw skills
- **testing_strategies.md** - Unit, integration, E2E testing approaches

## Distinction from Tools

**Automation/** focuses on **processes and workflows**, while **Tools/** focuses on **specific tool usage**.

Example:
```
✓ Automation/   "GitHub Actions workflow for pytest"
✗ Tools/   (not just tool usage)

✗ Automation/   "How to install pytest"
✓ Tools/   (installation guide)

✓ Automation/   "Building an OpenClaw skill for monitoring"
✗ Tools/   (not just configuration)
```
