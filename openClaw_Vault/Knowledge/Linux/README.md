# Knowledge/Linux

System administration, shell scripting, networking, and security notes for Linux environments.

## Category Definition

A note belongs in **Linux/** if it covers:
- System administration (users, permissions, services, systemd)
- Shell scripting (bash, zsh, POSIX sh)
- Networking (iptables, DNS, SSH, ports, firewall)
- Security (hardening, SELinux, AppArmor, access control)
- Filesystem and storage management
- Kernel and OS-level concepts

## Typical Contents

- Shell command references and scripting patterns
- Systemd service management
- Networking configuration and troubleshooting
- File permissions and ownership
- Process management
- Security hardening checklists

## Distinction from Tools

**Linux/** focuses on **OS-level concepts and administration**, while **Tools/** focuses on **application-level tool usage**.

```
✓ Linux/     "iptables firewall rules and chains"
✗ Tools/     (OS-level, not an application tool)

✓ Linux/     "bash parameter expansion and scripting"
✗ Tools/     (shell/OS level)
```
