# Knowledge/DataScience

Theory-focused data science notes including algorithms, statistical concepts, and mathematical foundations without code implementation.

## Category Definition

A note belongs in **DataScience/** if it discusses concepts **without code**:
- Statistical theory (variance, distributions, hypothesis testing)
- ML algorithms explained conceptually
- Mathematical formulas and proofs
- Data science principles and methodologies

## Typical Contents

- Algorithm explanations (decision trees, neural networks, clustering)
- Statistical concepts (mean, variance, correlation)
- Mathematical foundations (linear algebra, calculus for ML)
- Model evaluation metrics and theory
- Data science methodologies

## Example Notes

- **bias_variance_tradeoff.md** - Conceptual explanation of model complexity
- **clustering_algorithms.md** - K-means, DBSCAN theory without implementation
- **hypothesis_testing.md** - Statistical testing methodology

## NOT DataScience (Python Instead)

If a note includes **code examples** (even if explaining theory), it belongs in **Python/**.

Example:
```
✓ DataScience/   "Variance measures spread: σ² = Σ(x - μ)² / N"
✗ Python/   (no code)

✗ DataScience/   "variance = np.var(data)"
✓ Python/   (has code implementation)
```
